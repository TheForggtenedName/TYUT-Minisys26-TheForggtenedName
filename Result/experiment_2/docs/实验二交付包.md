## 快速开始

### 步骤 1：准备工程

```bash
# 使用实验一的 Vivado 工程
cd Minisys1_CPU

# 或创建新工程，参考 exper1deliver/README.md 的步骤 1-4
```

### 步骤 2：添加源文件

1. Flow Navigator → **Add Sources** → **Add or create design sources**
2. 添加以下新文件：
   - `sources/minisys/controlIO32.v`
   - `sources/minisys/digitube.v`
3. 替换以下文件：
   - `sources/minisys/memorio.v`
   - `sources/minisys/ioread.v`
   - `sources/minisys/minisys.v`

### 步骤 3：IP 核配置

参考实验一的步骤 4，保持 cpuclk, prgrom, ram 三个 IP 核不变。

### 步骤 4：综合验证

```
Flow Navigator → Run Synthesis
```

应无 Error（可能有 Warning，属正常）。

## 关键特性说明

### I/O 地址空间

| 设备 | 地址范围 | 控制信号 |
|------|---------|---------|
| LED (实验一) | 0xFFFFFC60-0xFFFFFC6F | LEDCtrl |
| Switch (实验一) | 0xFFFFFC70-0xFFFFFC7F | SwitchCtrl |
| **Digital Tube (实验二新增)** | **0xFFFFFC80-0xFFFFFC8F** | **DigitubeCtrl** |

### 数码管接口

```
地址 0xFFFFFC80:  写入第 0 位数字（低 4 位有效）
地址 0xFFFFFC81:  写入第 1 位数字（低 4 位有效）
地址 0xFFFFFC82:  同时写入两位数字
```

### 控制信号流向

```
CPU 执行 LW/SW 指令
  ↓
controlIO32 判断地址范围
  ├→ RAM 地址: MemRead/MemWrite = 1
  └→ I/O 地址: IORead/IOWrite = 1
  ↓
memorio 译码 I/O 地址
  ├→ LED: LEDCtrl = 1
  ├→ Switch: SwitchCtrl = 1
  └→ Digital Tube: DigitubeCtrl = 1
  ↓
外设 (leds, switchs, digitube) 响应
```

## 测试验证

### 仿真测试（可选）

编写测试程序：

```asm
# 写数码管显示"35"
li  $t0, 0xFFFFFC82
li  $t1, 0x0053        # 高 4 位=5, 低 4 位=3
sw  $t1, 0($t0)

# 读数码管状态
lw  $t2, 0($t0)
```

## 故障排查

### 综合错误

```
[Synth 8-344] Module 'controlIO32' not found
```

**解决**：确保 controlIO32.v 已添加到工程。

### 实现布线失败

```
[Place 30-574] Cannot place component
```

**解决**：确保芯片选为 `xc7a100tfgg484-1`（不是其他型号）。

### 数码管不显示

1. 检查 tube_sel 和 tube_digit_sel 是否有输出
2. 硬件引脚连接是否正确
3. 检查写数据是否到达 digitube 模块
