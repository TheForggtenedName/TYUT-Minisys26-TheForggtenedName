## 二、源代码文件详细说明

### 2.1 新建源代码文件

#### controlIO32.v（71行）
**位置**：`sources/minisys/controlIO32.v`  
**功能**：扩展的控制器，支持I/O寻址区分  
**关键特性**：
- 新增输入：`Alu_resultHigh[21:0]` - 用于地址范围判断
- 新增输出：`MemRead`, `IORead`, `IOWrite` - 区分内存/I/O访问
- 改进逻辑：`RegWrite` 排除 JR 指令
- 保持与 control32 的兼容性

**核心功能**：
```
地址判断 → 信号生成
0xFFFFFC00-0xFFFFFC7F → Is_IO_Access = 1 → IORead/IOWrite = 1
其他地址              → Is_IO_Access = 0 → MemRead/MemWrite = 1
```

#### digitube.v（137行）
**位置**：`sources/minisys/digitube.v`  
**功能**：七段数码管驱动模块  
**关键特性**：
- 支持两个4位七段显示器
- 动态扫描显示（~88Hz刷新率）
- 16种数字编码（0-9, A-F）
- 三种写入模式（单位写或同时写）
- 地址范围：0xFFFFFC80-0xFFFFFC8F

**核心实现**：
- 20位扫描计数器（2^18 = 262,144 时钟周期 ≈ 11ms）
- 两位数码管轮换选择
- 7段编码表初始化

### 2.2 修改源代码文件

#### memorio.v（45行）
**位置**：`sources/minisys/memorio.v`  
**修改内容**：
- 新增输出：`DigitubeCtrl` - 数码管芯片选择信号
- 新增地址译码：`0xFFFFFC80-0xFFFFFC8F → DigitubeCtrl = 1`
- 保持 LED 和 Switch 功能不变

#### ioread.v（32行）
**位置**：`sources/minisys/ioread.v`  
**修改内容**：
- 新增输入：`digituberead` - 数码管读使能
- 新增输入：`ioread_data_digitube[15:0]` - 数码管读数据
- 扩展多路选择器逻辑（从2路扩展到3路）
- 优先级：Switch > Digital Tube > 默认0x0000

#### minisys.v（250行）
**位置**：`sources/minisys/minisys.v`  
**修改内容**：
- 替换控制器：`control32` → `controlIO32`
- 新增信号声明：`memread`, `ioread_data_digitube`, `tube_sel`, `tube_digit_sel`, `digituberead`
- 更新 memorio 实例连接（新增 DigitubeCtrl）
- 更新 ioread 实例连接（新增两个参数）
- 新增 digitube 模块实例化
- 删除 `assign memread = memreg` 的旧逻辑

## 六、使用指南快速入门

### 6.1 集成到Vivado

```
1. 打开实验一的Vivado工程
2. Add Sources → Add or create design sources
3. 选择以下新文件：
   - controlIO32.v
   - digitube.v
4. 替换以下文件：
   - memorio.v
   - ioread.v
   - minisys.v
5. Run Synthesis（应无Error）
```

### 6.2 使用数码管

```asm
# 显示数字35
li $t0, 0xFFFFFC82         # 地址
li $t1, 0x0035             # 数据
sw $t1, 0($t0)             # 写入
```

### 6.3 常见问题解决

| 问题 | 解决方案 |
|------|---------|
| 数码管不显示 | 检查tube_sel/tube_digit_sel的硬件引脚连接 |
| 显示错误数字 | 验证digitube.v中的7段编码表 |
| 综合报错 | 确保controlIO32.v已添加到工程 |
| 某设备无响应 | 检查memorio.v中的地址译码条件 |
