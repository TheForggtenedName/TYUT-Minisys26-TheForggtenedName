# 实验二：Minisys-1 CPU 简单接口设计 - 实验报告

---

## 一、需求分析

### 1.1 设计目标

在实验一单周期 MIPS CPU 的基础上，完成 I/O 接口设计，特别是添加七段数码管显示外设的支持，实现完整的 I/O 地址空间管理和外设驱动。

### 1.2 功能需求

| 序号 | 功能模块 | 要求 | 实现状态 |
|------|---------|------|---------|
| 1 | I/O 寻址 | 区分内存和 I/O 访问，自动路由 | ✓ 完成 |
| 2 | 扩展控制器 | 新增 MemRead, IORead, IOWrite 信号 | ✓ 完成 |
| 3 | 地址译码 | LED/Switch/Digital Tube 自动区分 | ✓ 完成 |
| 4 | 数码管驱动 | 两个 4 位七段显示器，动态扫描 | ✓ 完成 |
| 5 | I/O 读多路选择 | 支持多设备读数据合并 | ✓ 完成 |
| 6 | 寄存器保护 | JR 指令排除 RegWrite | ✓ 完成 |

### 1.3 I/O 地址空间规划

| 设备 | 地址范围 | 大小 | 说明 |
|------|---------|------|------|
| LED（实验一继承） | 0xFFFFFC60-0xFFFFFC6F | 16B | 24位LED输出，两次写入 |
| Switch（实验一继承） | 0xFFFFFC70-0xFFFFFC7F | 16B | 24位开关输入，两次读取 |
| **Digital Tube（新增）** | **0xFFFFFC80-0xFFFFFC8F** | **16B** | **两个4位七段显示器** |

### 1.4 约束条件

- 工作时钟：23MHz（继承实验一）
- 指令集：MIPS 31条指令（继承实验一）
- 存储器：64KB ROM + 64KB RAM（继承实验一）
- 显示刷新率：≥44Hz（无闪烁感）

## 三、详细设计

### 3.1 controlIO32.v - 扩展控制器

**功能**：根据 ALU 结果的高位地址判断是否为 I/O 访问，生成相应的控制信号。

**输入**：
- `Opcode[5:0]` - 指令操作码
- `Function_opcode[5:0]` - R型指令的function码
- `Alu_resultHigh[21:0]` - ALU结果的高22位（用于地址判断）

**输出**：
- 基础信号：`RegDST`, `ALUSrc`, `MemorIOtoReg`, `RegWrite`, `Branch`, `nBranch`, `Jmp`, `Jal`, `Jrn`, `I_format`, `Sftmd`, `ALUOp` - 与control32相同
- **新增信号**：
  - `MemRead` - 从内存读（仅当地址不在I/O范围）
  - `IORead` - 从I/O读（仅当地址在I/O范围）
  - `MemWrite` - 写内存（仅当地址不在I/O范围）
  - `IOWrite` - 写I/O（仅当地址在I/O范围）

**关键逻辑**：

```verilog
// I/O 地址范围判断：0xFFFFFC00-0xFFFFFC7F
// 高 22 位全为 1 时，表示 I/O 访问
assign Is_IO_Access = (Alu_resultHigh[21:0] == 22'b1111111111111111111111) 
                      ? 1'b1 : 1'b0;

// 内存写：仅当为 SW 指令且地址不在 I/O 范围
assign MemWrite = Sw && !Is_IO_Access;

// I/O 写：仅当为 SW 指令且地址在 I/O 范围
assign IOWrite = Sw && Is_IO_Access;

// 内存读：仅当为 LW 指令且地址不在 I/O 范围
assign MemRead = Lw && !Is_IO_Access;

// I/O 读：仅当为 LW 指令且地址在 I/O 范围
assign IORead = Lw && Is_IO_Access;

// 改进的 RegWrite：排除 JR 指令
assign RegWrite = ((R_format || I_format || Lw || Jal) && !Jrn) 
                  ? 1'b1 : 1'b0;
```

**改进点**：
1. 自动区分内存和I/O访问，无需额外管理逻辑
2. JR指令不写寄存器，防止误操作
3. 保持与control32的兼容性（除新增信号外）

### 3.2 digitube.v - 七段数码管驱动

**功能**：驱动两个4位七段显示器，实现数字显示。

**输入**：
- `tube_clk` - 系统时钟 (23MHz)
- `tube_rst` - 复位信号（高电平有效）
- `tube_write` - 写使能（来自memorio的iowrite）
- `tube_cs` - 片选信号（来自memorio的DigitubeCtrl）
- `tube_addr[2:0]` - 地址偏移（alu_result[2:0]）
- `tube_wdata[15:0]` - 写数据（低4位有效）

**输出**：
- `tube_sel[7:0]` - 段选信号（a-g段 + 小数点）
- `tube_digit_sel[1:0]` - 位选信号（选择第0或第1位数码管）

**工作原理**：

1. **数据存储**：
```verilog
// 地址 0xFFFFFC80：写入第 0 位数字
if (tube_addr == 3'b000) digit_data_0 <= tube_wdata[3:0];

// 地址 0xFFFFFC81：写入第 1 位数字
if (tube_addr == 3'b001) digit_data_1 <= tube_wdata[3:0];

// 地址 0xFFFFFC82：同时写入两位
if (tube_addr == 3'b010) begin
    digit_data_0 <= tube_wdata[3:0];
    digit_data_1 <= tube_wdata[7:4];
end
```

2. **动态扫描**：
```verilog
// 20位扫描计数器
reg [19:0] scan_counter;
always @(posedge tube_clk) scan_counter <= scan_counter + 1;

// 每 2^18 个时钟切换一次
assign scan_sel = scan_counter[19:18];

// 在 23MHz 下：
// 切换频率 = 23MHz / 2^18 ≈ 88Hz
// 单位扫描周期 ≈ 5.6ms
```

3. **7段编码表**：
```verilog
seg_code[4'h0] = 8'b00111111;  // 0
seg_code[4'h1] = 8'b00000110;  // 1
seg_code[4'h2] = 8'b01011011;  // 2
seg_code[4'h3] = 8'b01001111;  // 3
seg_code[4'h4] = 8'b01100110;  // 4
seg_code[4'h5] = 8'b01101101;  // 5
seg_code[4'h6] = 8'b01111101;  // 6
seg_code[4'h7] = 8'b00000111;  // 7
seg_code[4'h8] = 8'b01111111;  // 8
seg_code[4'h9] = 8'b01101111;  // 9
seg_code[4'hA] = 8'b01110111;  // A
// ... (B-F)
```

**显示效果**：
- 两个数码管轮换扫描，约88Hz刷新
- 人眼观看呈现同时显示两位数字的效果
- 支持16进制数字显示（0-9, A-F）

### 3.3 memorio.v - 扩展的地址译码

**新增部分**：

```verilog
// 原有部分（保留）
assign LEDCtrl = ((iorw==1) && (caddress[31:4] == 28'hFFFFFC6)) ? 1'b1:1'b0;
assign SwitchCtrl = ((iorw==1) && (caddress[31:4] == 28'hFFFFFC7)) ? 1'b1:1'b0;

// 新增部分：数码管控制
assign DigitubeCtrl = ((iorw==1) && (caddress[31:4] == 28'hFFFFFC8)) ? 1'b1:1'b0;
```

**地址匹配原理**：
- LED：0xFFFFFC60-0xFFFFFC6F (bits[31:4] = 28'hFFFFFC6)
- Switch：0xFFFFFC70-0xFFFFFC7F (bits[31:4] = 28'hFFFFFC7)
- Digital Tube：0xFFFFFC80-0xFFFFFC8F (bits[31:4] = 28'hFFFFFC8)

### 3.4 ioread.v - 扩展的I/O读多路选择

**原实现**（仅支持Switch）：
```verilog
if(switchctrl == 1)
    ioread_data = ioread_data_switch;
else   
    ioread_data = ioread_data;
```

**扩展后**（支持Switch + Digital Tube）：
```verilog
if(switchctrl == 1)
    ioread_data = ioread_data_switch;
else if(digituberead == 1)
    ioread_data = ioread_data_digitube;
else   
    ioread_data = 16'h0000;
```

**新增输入**：
- `digituberead` - 数码管读使能
- `ioread_data_digitube[15:0]` - 数码管读数据

**核心改动**：

1. **替换控制器实例**：

```verilog

    // ... 原有连接
    .DigitubeCtrl(digituberead)          // 新增输出
```

3. **扩展ioread连接**：
```verilog
ioread multiioread(
    // ... 原有连接
    .digituberead(digituberead),         // 新增输入
    .ioread_data_digitube(ioread_data_digitube)  // 新增输入
```


## 五、用户手册

### 5.1 数码管写操作

**地址映射**：
- `0xFFFFFC80`：写入第 0 位数字（低 4 位）
- `0xFFFFFC81`：写入第 1 位数字（低 4 位）
- `0xFFFFFC82`：同时写入两位数字（高 4 位为第 1 位，低 4 位为第 0 位）

**汇编示例**：

```asm
# 显示数字 35
li    $t0, 0xFFFFFC82         # 基地址 + 2
li    $t1, 0x0035             # 高 4 位=3, 低 4 位=5
sw    $t1, 0($t0)             # 写入

# 显示十六进制 AB
li    $t0, 0xFFFFFC82
li    $t1, 0x00AB
sw    $t1, 0($t0)

# 分步写（先写第 0 位 = 7）
li    $t0, 0xFFFFFC80
li    $t1, 7
sw    $t1, 0($t0)

# 再写第 1 位 = 2
li    $t0, 0xFFFFFC81
li    $t1, 2
sw    $t1, 0($t0)
```

### 5.2 数码管读操作

**读取当前显示内容**：
```asm
li    $t0, 0xFFFFFC80
lw    $t2, 0($t0)             # 读入 $t2
```

**返回值**：总是 0x0000（当前版本不支持回读显示数据）

### 5.3 兼容性说明

#### LED 操作（保持不变）
```asm
# 写 LED（与实验一相同）
li    $t0, 0xFFFFFC60
li    $t1, 0xFFFF             # 24 位数据（低 16 位）
sw    $t1, 0($t0)
```

#### 开关读取（保持不变）
```asm
# 读开关（与实验一相同）
li    $t0, 0xFFFFFC70
lw    $t1, 0($t0)             # 读入 $t1
```

### 5.4 使用限制

- 数码管显示 0-9 和 A-F（16 种符号）
- 同一位显示只能是一个符号
- 显示刷新率 ~88Hz（充分快，无闪烁）
- I/O 操作延迟：<100ns（主要受内存访问延迟）
