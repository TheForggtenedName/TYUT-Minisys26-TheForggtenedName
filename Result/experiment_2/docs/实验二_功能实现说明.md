# 实验二：Minisys-1 CPU 简单接口设计

## 一、总体目标

在实验一 (单周期 MIPS CPU) 的基础上，完成 I/O 接口设计，特别是添加数码管（七段显示器）外设的支持。

---

## 二、主要修改和新增文件

### 1. controlIO32.v（新建）
**功能**：扩展的控制器，支持 I/O 寻址区分

**主要改进**：
- **新增输入**：`Alu_resultHigh[21:0]` —— 用于判断地址是否在 I/O 范围内
- **新增输出**：
  - `MemRead` —— 从内存读取数据（仅当地址不在 I/O 范围）
  - `IORead` —— 从 I/O 读取数据（仅当地址在 I/O 范围）
  - `IOWrite` —— 向 I/O 写入数据（仅当地址在 I/O 范围）
- **改名输出**：`MemtoReg` → `MemorIOtoReg`
- **修改逻辑**：
  - `RegWrite` 现在排除 JR 指令（防止 JR 指令误写寄存器）
  - 地址判断：I/O 范围为 `0xFFFFFC00-0xFFFFFC7F`（Alu_resultHigh[21:0] == 22'b1111...）

**工作原理**：
```
I/O 地址范围判断：
- 仅当 Alu_resultHigh[21:0] 全为 1 时，表示 I/O 访问
- 此时 MemRead/MemWrite 被强制为 0，IORead/IOWrite 被激活
- 否则为普通内存访问
```

### 2. digitube.v（新建）
**功能**：七段数码管显示驱动

**特性**：
- **显示方案**：两个 4 位七段显示器
- **基地址**：`0xFFFFFC80-0xFFFFFC8F`
- **控制方式**：
  - 地址 `0xFFFFFC80`：写入第 0 位数字（低 4 位）
  - 地址 `0xFFFFFC81`：写入第 1 位数字（低 4 位）
  - 地址 `0xFFFFFC82`：同时写入两位数字（低 4 位写第 0 位，高 4 位写第 1 位）

**硬件接口**：
- `tube_sel[7:0]` —— 段选信号 (a-g 段 + 小数点)
- `tube_digit_sel[1:0]` —— 位选信号（区分两个数码管）

**动态扫描**：
- 扫描计数器频率约为系统时钟的 1/2^20
- 实现两个数码管的快速轮换显示

**7 段编码表**（高位到低位）：
| 数字 | 编码 | 显示的段 |
|------|------|---------|
| 0    | 00111111 | a,b,c,d,e,f |
| 1    | 00000110 | b,c |
| 2    | 01011011 | a,b,d,e,g |
| 3    | 01001111 | a,b,c,d,g |
| ... | ... | ... |
| F    | 01110001 | a,e,f,g |

### 3. memorio.v（修改）
**修改内容**：添加数码管芯片选择信号

**新增输出**：`DigitubeCtrl`

**地址译码**：
```verilog
// LED 地址范围：0xFFFFFC60-0xFFFFFC6F
assign LEDCtrl = ((iorw==1) && (caddress[31:4] == 28'hFFFFFC6)) ? 1'b1:1'b0;

// Switch 地址范围：0xFFFFFC70-0xFFFFFC7F
assign SwitchCtrl = ((iorw==1) && (caddress[31:4] == 28'hFFFFFC7)) ? 1'b1:1'b0;

// 数码管地址范围：0xFFFFFC80-0xFFFFFC8F（新增）
assign DigitubeCtrl = ((iorw==1) && (caddress[31:4] == 28'hFFFFFC8)) ? 1'b1:1'b0;
```

---

### 4. ioread.v（修改）
**修改内容**：扩展 I/O 读数据多路选择器

**新增输入**：
- `digituberead` —— 数码管读使能
- `ioread_data_digitube[15:0]` —— 数码管读数据

**多路选择逻辑**：
```verilog
if(ior == 1) begin
    if(switchctrl == 1)
        ioread_data = ioread_data_switch;
    else if(digituberead == 1)
        ioread_data = ioread_data_digitube;
    else   
        ioread_data = 16'h0000;
end
```

### 5. minisys.v（修改）
**修改内容**：集成新的控制器和数码管模块

**关键改动**：
1. **替换控制器**：`control32` → `controlIO32`
   - 新增连接 `Alu_resultHigh` 输入
   - 新增连接 `MemRead`, `IORead`, `IOWrite` 输出
   
2. **更新 memorio 实例**：
   - 新增连接 `DigitubeCtrl` 输出

3. **更新 ioread 实例**：
   - 新增连接 `digituberead` 和 `ioread_data_digitube`

4. **新增数码管实例**：
   ```verilog
   digitube tube(
       .tube_clk(clock),
       .tube_rst(rst),
       .tube_write(iowrite),
       .tube_cs(digituberead),
       .tube_addr(alu_result[2:0]),
       .tube_wdata(write_data[15:0]),
       .tube_sel(tube_sel),
       .tube_digit_sel(tube_digit_sel)
   );
   ```

5. **新增信号声明**：
   - `wire memread` —— 从 controlIO32 输出
   - `wire [15:0] ioread_data_digitube` —— 数码管读数据
   - `wire [7:0] tube_sel` —— 段选输出
   - `wire [1:0] tube_digit_sel` —— 位选输出
   - `wire digituberead` —— 数码管读使能

## 三、I/O 地址空间分配

| 设备 | 地址范围 | 大小 | 说明 |
|------|---------|------|------|
| LED | 0xFFFFFC60-0xFFFFFC6F | 16B | 24 位 LED，分两次写入 |
| Switch | 0xFFFFFC70-0xFFFFFC7F | 16B | 24 位开关，分两次读取 |
| Digital Tube | 0xFFFFFC80-0xFFFFFC8F | 16B | 两个 4 位七段显示器 |

## 四、使用示例

### 写数码管
```asm
# 将数字 3 写入第 0 位，数字 5 写入第 1 位
li $t0, 0xFFFFFC82      # 基地址 + 偏移 2
li $t1, 0x0053          # $t1 = 0x0053 (高 4 位 = 5, 低 4 位 = 3)
sw $t1, 0($t0)          # 写入两位数字
```

### 读数码管
```asm
lw $t2, 0($t0)          # 从 0xFFFFFC80 读取当前显示状态
```

## 七、常见问题

**Q: 为什么要用 Alu_resultHigh 而不是完整的地址？**
A: 节省 controlIO32 的输入端口数。只需高 22 位即可判断 I/O 范围，低 10 位留给外设内部地址译码。

**Q: 数码管如何实现动态扫描？**
A: 通过扫描计数器每 2^18 个时钟周期切换一次位选，人眼无法感知闪烁。

**Q: 能否修改 I/O 基地址？**
A: 可以，但需同时修改 controlIO32.v 的地址判断逻辑和 memorio.v 的芯片选择逻辑。
