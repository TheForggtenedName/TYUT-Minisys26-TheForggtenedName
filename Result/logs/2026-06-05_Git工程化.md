[上午 09:30] 在 Vivado 综合日志中发现 Critical Warning：
  [Synth 8-4442] BlackBox module cpuclk has unconnected pin reset
  原因：cpuclk (Clocking Wizard) 有 5 个端口 -- clk_in1, clk_out1, clk_out2, reset, locked，
  但 minisys.v 实例化时只连了前 3 个。reset 是输入端口，悬空引发 Critical Warning。
  修复：.reset(1'b0) 接地 + .locked() 显式留空。

[下午] 开始逐块学习 minisys.v 顶层模块代码。

  第 1 块 — 顶层端口（7 个物理引脚）：
    fpga_clk(100MHz 晶振)、fpga_rst(复位按键)、switch2N4(24 拨码)、led2N4(24 LED)、
    start_pg(UART 烧写键)、rx/tx(串口收发)
    input = 外部信号进来，只能读 → 类比：教室门口的签到机
    output = FPGA 给值出去 → 类比：投影仪
    [23:0] = 24 根物理铜线捆成一束，类比：IDE 排线，不是数组

  学习要点：
  - 端口名必须与 minisys.xdc 中的引脚约束一一对应
  - module ≠ C++ struct：struct 是内存中用完即释放，module 是硅片上一直通电运行
  - 所有 always/assign 并行执行，不存在先后顺序

  下一步学习内容：第 2 块 wire 声明（CPU 内部数据线）和第 4 块 6 个模块实例化
