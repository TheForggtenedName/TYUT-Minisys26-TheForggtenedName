    修复前：assign Zero = (ALU_output_mux[31:0] == 32'h00000000) ? 1'b1 : 1'b0;
    修复后：assign Zero = (ALU_Result == 32'h00000000) ? 1'b1 : 1'b0;

  根因分析：
    ALU_output_mux 是 case 电路中 8 种 ALU 运算的原始结果（中间值）。
    ALU_Result 在此基础上还经过 SLT/LUI/Shift 三路 MUX 最终选择（最终值）。
    三类指令下两者不同：
      SLT/SLTI: ALU_output_mux = 减法结果, ALU_Result = 0 或 1 → Zero 判断反了
      LUI:      ALU_output_mux = NOR结果, ALU_Result = {B[15:0], 16'b0} → 两者可能不等
      Shift:    ALU_output_mux = 默认case值, ALU_Result = Sinput → 完全不同

  暴露条件：只有 BEQ/BNE 前一条是 SLT/LUI/移位时才出错。
  实验一测试程序分支前恰好是 ADDI（常规 ALU，ALU_Result == ALU_output_mux）→ 没暴露。
  这是第三个"凑巧不暴露"的自引入 Bug，新增为问题 #9。

[晚上 21:30] 以实验一功能实现说明文档为基础，逐概念深入问答学习：

Q1: IP核是干什么的？
  答：IP 核是 Vivado 的硬件知识产权模块，Xilinx 官方预封装的硬件电路"黑盒"。
  本设计三个 IP 核：
  - cpuclk (Clocking Wizard)：MMCM 锁相环分频，100MHz→23MHz(CPU)+10MHz(UART)
  - prgrom (Block Memory ROM)：32位×16384（64KB）程序只读存储器，上电从 .coe 加载
  - ram (Block Memory RAM)：32位×16384（64KB）数据读写存储器
  三者都用 FPGA 片内硬核资源（MMCM + BRAM），不需要外挂芯片。

Q2: cpuclk 100MHz 怎么分成 23MHz 和 10MHz？数量上对不上啊？
  答：MMCM 不是简单除以整数，而是先乘后除的链式分频：
  100MHz / 5 = 20MHz（参考频率）
  20MHz × 46 = 920MHz（VCO 内部高频）
  920MHz / 40 = 23MHz（CPU 时钟）
  920MHz / 92 = 10MHz（UART 时钟）
  FPGA 的 PLL/MMCM 硬件天生支持这种小数分频，不是写个计数器来分频。

Q3: UART 又是什么？
  答：通用异步收发传输器，串行通信协议。项目里通过 USB 转串口把编译好的 MIPS
  机器码烧写到 FPGA 的 ROM/RAM，省去每次修改程序都要重新跑综合→实现→生成
  比特流的冗长流程（10 分钟变成十几秒）。对应模块是 upg.v/uart_bmpg.v，
  其中 uart_bmpg.edif 是 Xilinx 提供的加密网表文件。

Q4: "32位×16384" 中 32 和 16384 分别代表什么？64KB 又是啥？
  答：
  - 32 位 = 数据位宽（每次读写操作取出 32 位数据）
  - 16384 = 地址深度（存储单元个数，地址线能索引到 16384 个不同位置）
  - 64KB = 总容量（16384 × 4 byte = 65536 byte）
  三者是从不同视角描述同一块存储器。

Q5: 存储设计中说的"路由"是什么？
  答：memorio.v 模块的核心功能——根据 CPU 发出的地址，把读写请求分发到正确的设备。
  地址高 22 位判断逻辑：
  - 全为 0（0x00000000~0x0000FFFF）→ 走 RAM（真正的数据存储器）
  - == 0x3FFFFC6（0xFFFFFC60~0xFFFFFC6F）→ 走 LED 模块
  - == 0x3FFFFC7（0xFFFFFC70~0xFFFFFC7F）→ 走拨码开关模块
  同时管读数据的返回方向——memread=1 选 RAM 输出给 CPU，否则选 IO 设备输出。

今日学习收获：
  - IP 核本质：Vivado 黑盒硬件模块，配置参数即可用，底层是 FPGA 硬核
  - MMCM 分频原理：先乘法飙到高频(VCO)，再除法分给各路，不是简单的 ÷N 计数器
  - UART 在项目中的角色：快速烧写程序的串口通道，避免每次重新跑综合实现
  - 存储器参数：32 位=数据宽，16384=地址深，64KB=总容量，三者视角不同
  - Memory-Mapped I/O 路由：地址高位译码区分不同设备，一条 LW/SW 就能操作外设
