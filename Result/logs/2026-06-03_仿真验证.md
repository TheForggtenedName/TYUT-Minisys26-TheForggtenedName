**新发现问题 #12 — IP 核被锁定（cpuclk/prgrom/ram）**：

  每次打开工程都弹出的 Warning：
    WARNING: [IP_Flow 19-2162] IP 'cpuclk' is locked:
    * Current project part 'xc7a100tfgg484-1' and the part
    'xc7a100tcsg324-1' used to customize the IP 'cpuclk' do not match.

  原因：这是之前问题 #10（芯片封装选错 CSG324）的后遗症。
  三个 IP（cpuclk/prgrom/ram）当初是按 CSG324 生成的，后来工程芯片改成
  FGG484，但 IP 还是旧的。Vivado 检测到芯片不匹配就把 IP 锁定了。

  影响分析：
  - 仿真不受影响：仿真用的是独立的功能网表（cpuclk_sim_netlist.v 等），
    不依赖 IP 综合结果
  - 综合/实现受影响：IP 被锁后 Vivado 使用过期的 synthesis checkpoint，
    即旧芯片的网表，最终 bitstream 对应的是错误芯片，不可靠

  解决方案 — 在 Vivado 中升级 IP：
    步骤 1：菜单 Tools → Report → Report IP Status
    步骤 2：弹出的 IP Status 窗口列出 3 个 Locked IP
            （Status 列显示 "IP locked"），全选
    步骤 3：点击底部 "Upgrade Selected" 按钮
    步骤 4：Vivado 自动为当前芯片（FGG484）重新生成 IP，
            等待 1-2 分钟完成
    步骤 5：升级完成后重新 Run Synthesis → Run Implementation
            → Generate Bitstream

  原理：IP 锁定是 Vivado 的安全机制。芯片封装不同意味着 FPGA 内部的
  MMCM/BUFG/BRAM 位置不同，CSG324 和 FGG484 虽然芯片型号同为 xc7a100t，
  但 Die 上的硬件资源坐标不同。Upgrade IP 会重新调用 IP 生成器，
  根据新的芯片封装重新计算资源映射。

**新发现问题 #13 — launch_runs 缓存脚本芯片锁定**：

  IP 升级完成后执行 launch_runs synth_1，综合日志中显示：
    Command: synth_design -top minisys -part xc7a100tcsg324-1
    Loading part: xc7a100tcsg324-1
  而此时 get_property part [current_project] 返回 xc7a100tfgg484-1。

  升级前的情况：IP=CSG324, 工程=FGG484 → 锁 Warning
  升级后的诡异现象：IP=FGG484, 工程=FGG484，但 launch_runs 脚本里芯片=CSG324 → 锁 Warning

  原因：launch_runs 使用的 minisys.tcl 是 Vivado 缓存文件，里面 -part 参数
  被写死为最初创建工程时的芯片（CSG324）。reset_run 只清理运行结果，
  不重新生成 .tcl 脚本。所以无论怎么 reset_run + launch_runs，
  永远用的是旧芯片。

  解决：绕过 launch_runs 机制，在 GUI Tcl Console 中逐步执行流水线命令：
    synth_design -top minisys -part xc7a100tfgg484-1  (直接在 GUI 中跑综合)
    opt_design                                           (逻辑优化)
    place_design                                         (布局)
    route_design                                         (布线)
    write_bitstream -force <路径>/minisys.bit             (生成比特流)

  这 5 条命令等价于 Vivado 的 Run Synthesis → Run Implementation →
  Generate Bitstream 全流程，只是不依赖缓存脚本。缺点是每次重启 Vivado 后
  需要重新从 synth_design 开始。

**新发现问题 #14 — write_bitstream 缺少文件路径报错**：

  执行 write_bitstream -force 时报错：
    ERROR: [Common 17-163] Missing value for option 'file'

  原因：write_bitstream 的 -file 参数不是可选的。-force 只是覆盖标志，
  后面必须紧跟目标 .bit 文件的完整路径。

  解决：write_bitstream -force D:/.../impl_1/minisys.bit
