**严重潜伏 Bug #6 在此引入（见后文详细分析）**：
  目标寄存器选择用了 RegDst=1→rd, RegDst=0→rt 的二选一。
  JAL 指令无 rd 无 rt 字段，MIPS 规定隐式写 $31($ra)。
  但 RegDst=0 时选了 instruction[20:16]（跳转目标地址的其中 5 位）作为目标寄存器号！
  幸运的是测试程序的 JAL 目标地址 bit[20:16] 恰好全为 0 → 指向 $0 → 写保护生效，
  数据写入被 $0 丢弃。这个 Bug 在仿真中完全不可见！直到人工审查才被发现。

  但 JR 只跳转不写寄存器！幸好 JR 指令编码中 rd 字段恰好是 00000，

[13:00] 补写 executs32.v（执行单元）：

  第46-51行 移位操作补全：
    3'b000: Sinput = Binput << Shamt;                          // SLL
    3'b010: Sinput = Binput >> Shamt;                          // SRL
    3'b100: Sinput = Binput << Ainput[4:0];                    // SLLV
    3'b110: Sinput = Binput >> Ainput[4:0];                    // SRLV
    3'b011: Sinput = $signed(Binput) >>> Shamt;                // SRA
    3'b111: Sinput = $signed(Binput) >>> Ainput[4:0];          // SRAV

  第59行 SLT/SLTI 补全：
    ALU_Result = ($signed(Ainput) < $signed(Binput)) ? 32'd1 : 32'd0;

  第60行 LUI 补全：
    ALU_Result = {Binput[15:0], 16'b0};

  第61行 移位结果补全：
    ALU_Result = Sinput;

  第71-78行 ALU 8 种运算补全：
    3'b000: ALU_output_mux = Ainput & Binput;       // AND
    3'b001: ALU_output_mux = Ainput | Binput;       // OR
    3'b010: ALU_output_mux = Ainput + Binput;       // ADD/ADDI
    3'b011: ALU_output_mux = Ainput + Binput;       // ADDU
    3'b100: ALU_output_mux = Ainput ^ Binput;       // XOR
    3'b101: ALU_output_mux = ~(Ainput | Binput);    // NOR
    3'b110: ALU_output_mux = Ainput - Binput;       // SUB
    3'b111: ALU_output_mux = Ainput - Binput;       // SUBU

  学习发现：SRA（算术右移）用 `$signed(Binput) >>> Shamt`，高位填符号位；
  SRL（逻辑右移）用 `Binput >> Shamt`，高位填 0。Verilog 中 >> 和 >>> 的
  行为取决于左操作数是否声明为 signed。SRA 必须用 $signed() 包裹。

**记录错误 #7（仿真时间不够—PLL 未锁定 + reset 未释放）**：
  问题分两层：
  (1) cpuclk IP (Clocking Wizard) 是 MMCM/PLL，上电后需要锁定时间。
     Vivado 仿真中 PLL 锁定约需 5-7μs（5000ns-7000ns）。在此之前 clock 输出为 0。
  (2) minisys_sim.v 第40行：#7000 rst = 0; — CPU 在 7000ns 之前一直处于复位状态。

  所以前 7000ns CPU 根本就不工作！默认仿真只跑 1000ns，当然什么都看不到。

  解决方案：Tcl Console 输入 run 20000ns → 仿真延长到 20000ns：
    时间 0~7000ns:   clock=0 (PLL 锁定中), rst=1 (复位)
    时间 7000ns:      rst=0 (复位释放)
    时间 7000ns 后:   clock 开始有波形 (PLL 锁定完成)
    时间 20000ns:     clock=1, pc_plus_4=0x78, instruction=0x341d001d (有效指令)
    CPU 成功执行了约 30 条 ORI 指令！

  经验：PLL/MMCM 的仿真行为与组合逻辑不同，锁相环需要时间，这个在纯数字逻辑
  仿真中容易被忽略。

  --- 我们自己引入的逻辑 Bug（凑巧没暴露） ---
  #7  JAL 写回目标寄存器地址错误（潜伏 Bug）
      JAL 没有 rt/rd 字段，MIPS 规定隐式写 $31($ra)
      但代码中 RegDst=0 选了 instruction[20:16]（跳转地址的其中 5 位）
      凑巧测试程序 bit[20:16]=00000 → $0 写保护 → 写入被丢弃
      这个 Bug 仿真完全看不出来！必须修改：
        if (Jal == 1'b1)
            write_register_address = 5'b11111;  // 显式写 $31

  #8  JR 指令 RegWrite 未排除（潜伏 Bug）
      JR 属于 R 型(opcode=000000) → R_format=1 → RegWrite=1
      但 JR 只跳转不写寄存器！
      凑巧 JR 编码中 rd=00000 → 写 $0 → 被写保护
      必须修改：RegWrite = ((R_format && !Jrn) || I_format || ...)

      创建工程时选了 Nexys4 的 CSG324，约束文件对应 FGG484
      Implementation 阶段 20+ 引脚全部不匹配
      解决：改芯片为 xc7a100tfgg484-1，重新综合+实现

  #11 全部 .v 文件中文注释乱码
